> **37.5% מהלידים ב-Salesforce רשומים ב-"none".
> פירוש מעשי: מתוך 8 לידים חודשיים — 3 לידים טסים בעיוורון. אין ROI. אין תקציב מבוסס-נתונים. אין צמיחה חכמה.**

## ⚙️ חלק ב׳ — Validation Rule: חוסם שמירה ללא מקור

### להוסיף ב-Salesforce → Lead Object → Validation Rules

**שם הכלל:** `Lead_Source_Required_UPE`

```apex
AND(
  ISBLANK(LeadSource),
  NOT(ISPICKVAL(Status, "Junk"))
)
```

**הודעת שגיאה (Hebrew):**
> `חובה לבחור מקור ליד לפני שמירה. בחר ערך מהרשימה: Facebook/Instagram, LinkedIn, YouTube, Referral, Website, Cold Outreach, Event, Email Campaign.`

**הודעת שגיאה (English fallback):**
> `Lead Source is required. Please select the channel that generated this lead before saving.`

> 💡 **הערת Admin:** הכלל חל על כל ה-Profiles פרט ל-System Administrator (כדי לאפשר backfill של רשומות ישנות).

## 🔄 חלק ג׳ — Flow אוטומטי: מילוי אוטומטי מ-UTM

פרמטרי UTM (utm_source, utm_medium, utm_campaign) לוכדים את ההקשר של הקמפיין מהתנועה לאתר — אם לא לוכדים אותם בעת שליחת טפסים ומיפוים לשדות Salesforce, מאבדים את נתוני ה-Attribution.

### מיפוי UTM → Lead Source (Salesforce Flow / Process)

```
אם utm_source = 'facebook' או 'instagram' OR utm_medium = 'paid-social-meta'
   → Lead Source = 'Facebook / Instagram'
   → Lead Source Detail = [utm_campaign value]

אם utm_source = 'linkedin' OR utm_medium = 'paid-social-linkedin'
   → Lead Source = 'LinkedIn'
   → Lead Source Detail = [utm_campaign value]

אם utm_source = 'youtube' OR utm_medium = 'video'
   → Lead Source = 'YouTube'
   → Lead Source Detail = [utm_campaign value]

אם utm_medium = 'email'
   → Lead Source = 'Email Campaign'
   → Lead Source Detail = [utm_campaign value]

אם utm_source = 'organic' OR (ISBLANK(utm_source) AND referrer contains 'google.com')
   → Lead Source = 'Website – Organic'
```

### Web-to-Lead Hidden Fields להוסיף לכל טופס ב-UPE:

```html
<!-- Hidden UTM fields — add to ALL UPE web forms -->
<input type="hidden" id="utm_source" name="utm_source" value="">
<input type="hidden" id="utm_medium" name="utm_medium" value="">
<input type="hidden" id="utm_campaign" name="utm_campaign" value="">
<input type="hidden" id="utm_content" name="utm_content" value="">

<script>
  // Auto-populate UTM fields from URL parameters
  (function() {
    const params = new URLSearchParams(window.location.search);
    ['utm_source','utm_medium','utm_campaign','utm_content'].forEach(function(key) {
      const el = document.getElementById(key);
      if (el && params.get(key)) {
        el.value = params.get(key);
      }
    });
  })();
</script>
```

## ⚡ חלק ז׳ — Backfill: מה לעשות עם 37.5% הרשומות הקיימות?

הפתרון לרשומות ישנות: backfill שלהן עם "Unknown" או "Historical" כדי שלא ידרסו ערכים בעת המרת לידים חדשים — זהו מלכודת נפוצה ב-Salesforce.

**שלבי Backfill (Admin — Data Loader):**

```
1. Export: כל Leads עם LeadSource = null, Created Date < [תאריך הפעלת הכלל]
2. Update: LeadSource = "Unknown – Backfill"
3. Import חזרה ב-Data Loader
4. הוסף Filter ל-Dashboard: הוצא "Unknown – Backfill" מניתוח ה-ROI השוטף
```

> 💡 רשומות מסוג "Unknown – Backfill" שומרות על שלמות הנתונים ההיסטוריים מבלי "לזהם" את דיווח ה-ROI העתידי.

## 🔴 חלק ח׳ — הגנה: למה זה דחוף יותר ממה שנראה

כשהמקור רשום "Web" בגלל שזה מה שהיה בטופס הבקשה — הווובינר, הסרטון, מודעת ה-LinkedIn, שלושת בעלי העניין שביקרו בדף המחירים — כולם בלתי נראים. זהו ה-Blind Spot של Salesforce, וזו הסיבה שרוב צוותי שיווק B2B לא יכולים להוכיח מה מניע את ה-Pipeline.

**מה UPE מסתכנת אם לא מתקנת:**
- ❌ תקציב Meta Ads מוגדל על בסיס "נראה שעובד" — בלי הוכחה
- ❌ LinkedIn מקבל פחות תקציב כי "לא רואים ממנו לידים" — אך הוא מייצר אבל לא מתויג
- ❌ Referrals לא ממונפים כי לא יודעים כמה הם שווים
- ❌ בפגישות תקציב רבעוניות — אין data להגן על תקציב שיווק

**מה UPE מרוויחה אחרי התיקון:**
- ✅ יודעים בדיוק כמה כל ליד עולה לפי ערוץ
- ✅ יכולים להכפיל תקציב בערוץ שמייצר הכי הרבה לידים איכותיים
- ✅ ראיות ברורות לכיוון קמפיינים עתידיים
- ✅ כל איש מכירות רואה את ה-Journey המלא של הליד שלו
